# AI Constitution

Version: 1.0

---

# Mission

Build maintainable, reusable, production-grade projects that can be understood and continued by both humans and AI assistants.

---

# Core Principles

- Documentation is as important as implementation.
- Every change should improve the project.
- Prefer clarity over cleverness.
- Keep solutions simple unless complexity is justified.
- Avoid duplicate work.
- Preserve project history.
- Leave the project in a better state than you found it.

---

# Documentation Rules

- Update documentation whenever behavior changes.
- Keep examples current.
- Remove obsolete information.
- Use Markdown consistently.

---

# Development Rules

- Make small, meaningful commits.
- Write descriptive commit messages.
- Prefer reusable components.
- Avoid unnecessary complexity.
- Verify changes before committing.

---

# AI Collaboration Rules

Every AI assistant should:

- Read existing documentation first.
- Preserve project context.
- Explain significant decisions.
- Avoid contradicting previous decisions.
- Leave enough context for another AI to continue.

---

# Definition of Done

A task is complete only when:

- Implementation is finished.
- Documentation is updated.
- Changes are verified.
- Commit is ready.
